# Supabase Integration for GoFile Upload Tracking

This integration stores GoFile upload records (streamer name, link, and date) in Supabase.

## Prerequisites

1. **Docker Desktop** - Required for local Supabase development
   - Download: https://www.docker.com/products/docker-desktop
   - Verify: `docker version`

2. **Supabase CLI** - Install via npm or other package manager
   ```bash
   npm install -g supabase
   # or
   brew install supabase/tap/supabase
   ```

## Setup Instructions

### 1. Initialize Supabase (Local Development)

```bash
# Initialize Supabase in your project
supabase init --yes

# Start local Supabase stack
supabase start
```

After starting, you'll see output with your local credentials:
- API URL: `http://127.0.0.1:54321`
- Anon key: (copy this for configuration)

### 2. Create the Database Table

Run the SQL migration to create the `gofile_uploads` table:

```bash
supabase db reset --local
```

Or manually execute the SQL in `supabase/migrations/create_gofile_uploads_table.sql`

### 3. Configure GoOnDVR

Add the Supabase credentials when running GoOnDVR:

```bash
./goondvr \
  --enable-gofile-upload \
  --supabase-url "http://127.0.0.1:54321" \
  --supabase-api-key "YOUR_ANON_KEY_HERE"
```

Or set via environment variables in your deployment.

## Table Schema

The `gofile_uploads` table stores:
- `id` - Auto-incrementing primary key
- `streamer_name` - Name of the streamer
- `gofile_link` - GoFile download URL
- `upload_date` - Timestamp of the upload

## Production Deployment

For production, use a hosted Supabase project:

1. Create a project at https://supabase.com
2. Get your project URL and anon/service key from Settings > API
3. Use those credentials instead of local ones

## Querying Upload Records

You can query the data via:
- Supabase Dashboard: http://127.0.0.1:54323 (local)
- SQL queries through the Supabase client
- REST API endpoints automatically generated by Supabase
